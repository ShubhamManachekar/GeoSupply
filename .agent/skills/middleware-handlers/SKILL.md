# Middleware Handlers Skill (FA v2)

## Context
The `EventBus` handles all asynchronous communication and lateral routing across the swarm. The rules here dictate how to govern this middleware logically without generating traffic mock buffers.

## EventBus G3 Signing Protocol
-   Every message fired on the bus must contain an HMAC-SHA256 signature generated by the agent's unique access key (`event_bus.py`).
-   The `SecurityAgent` manages 30-day key rotations. Middleware tests MUST verify that payloads signed with a 31-day old key are strictly rejected by the `EventBus` receiver queue.
-   **No Bypass Rules**: Do not allow unsigned wildcard inputs onto the bus unless strictly flagged for `Tier 0` ingestion bypass processing.

## Handler Exception Tolerance
-   When multiple subagents are subscribed to the "intelligence_feed" topic, an exception raised inside one subagent's handler MUST NOT crash the publish queue.
-   Test this constraint logically: Fire a bad event to a bad handler, verify it errors out, and map an assertion that the secondary good handler still executed successfully (`True`).

## Back-Pressure and Memory Limits
-   With 17 ingestion APIs active (Phase 15 additions), message flooding is a risk (R7).
-   Limit queue buffers to absolute maximums of 5000 cached events inline. Unprocessed events must spill over to SQLite disk journaling to protect host memory. Provide `@timeout` decorators on slow EventBus consumers.
