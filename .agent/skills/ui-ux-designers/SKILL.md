# UI/UX Designers Skill (FA v2)

## Context
The GeoSupply Admin Portal (Phase 9) exposes the massive underlying intelligence pipeline output into human-consumable formats. We do not mock data to test renders.

## Multipage Lazy Loading
-   **CRITICAL CONSTRAINT**: The `KnowledgeGraphAgent` can yield >100K entities.
-   Do **NOT** load the entire dataset into Pandas/Streamlit on boot.
-   Use `st.cache_data` and Streamlit's new multipage lazy loading (`st.navigation()`, `st.Page()`). Ensure that querying the SQLite databases pushes compute strictly to SQL `LIMIT/OFFSET`.

## Admin Overrides
-   The "Override Pattern" UI must let admins inject manual intelligence suppressing false positive anomalies generated by `OverridePatternSubAgent`.
-   The UI must explicitly indicate if data is `LIVE` (green), `DEGRADED` (yellow), or `STALE` (red) based on the current cache TTL from `HealthCheckAgent`. 
-   Aesthetically, the UI needs to reflect a dashboard layout—use `st.columns()` effectively to segment raw feeds versus aggregated event timelines.

## Testing the UI
-   No Selenium mock click testing is required under current rules, but Streamlit's `AppTest` framework **SHOULD** be run natively against the backend SQLite cache.
